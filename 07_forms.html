Form structure

  • The <form> element
    • is mandatory for all forms
    • should include anything that belongs to the form, also titles, help texts, error messages etc...
    • should probably have a submit button at the end
    <form>
  <h3>Registration form</h3>
  ...
  <p>fields with an * are required</p>
  ...
  <p>
    after submitting the form, you'll receive an email
    with further instructions
  </p>
  <p><input type="submit" value="Send"></p>
</form>
  • → markup suggestion: group each label/input pair in a <p>

[Fieldsets]

  • If your form contains more than a few controls, use <fieldset> and <legend> to create groups: 
    <form action="#">
  ...
  <fieldset>
    <legend>Login information</legend>
    ...
  </fieldset>
  <fieldset>
    <legend>Personal information</legend>
    ...
  </fieldset>
  <fieldset>
    <legend>Preferences</legend>
    ...
  </fieldset>
  ...
</form>

Value and id attributes

  • Each control may have value and id attributes. Don't worry too much about their exact meaning. Just remember for now:
    • basically, the value attribute is what will be sent with the form
    • a group of radiobuttons should have the same value attribute
    • the value attribute also sets the value of a field or text on a button
    • the for attribute of <label> should point to the id attribute of the corresponding control:
    <p>
  <label for="name">Your name:</label>
  <input type="text" id="name">
</p>

Labels (1)

  • All controls should be preceded by their <label>: 
    <p>
  <label for="name">Your name:</label>
  <input type="text" id="name">
</p>
  • Exception: radionbuttons and checkboxes can be nested in their <label>; for and id are not needed here: 
    <p>
  <label><input type="radio" name="gender" value="f" checked> Female</label>
  <label><input type="radio" name="gender" value="m"> Male</label>
</p>
  • → following these rules will help future layout with CSS

Labels (2)

  • Always provide labels, even if they aren't visible in the design:
    • labels helps screen readers; we will hide them later with CSS

Input types

Text input types

  • Text field: 
    <label for="fname">First name:</label>
<input type="text" id="fname" placeholder="bv. Bob">s
  • Text area: 
    <label for="bio">Bio:</label>
<textarea id="bio" cols="40" rows="5" required></textarea>
  • Password field: 
    <label for="pw">Password:</label>
<input type="password" id="pw" placeholder="at least 8 characters">
  • Hidden field (used to send extra data): 
    <input type="hidden" name="action" value="xyz">

Radiobuttons & checkboxes

  • Radiobuttons: 
    <label><input type="radio" name="group1" value="a"> option a</label>
<label><input type="radio" name="group1" value="b"> option b</label>
<label><input type="radio" name="group1" value="c"> option c</label>
    → note: radiobuttons of the same group have identical name attributes
  • Checkboxes: 
    <label><input type="checkbox" value="a"> choice 1</label>
<label><input type="checkbox" value="b"> choice 2</label>
<label><input type="checkbox" value="c"> choice 3</label>
    → note: checkboxes have identical name attributes

Lists

  • Dropdown list: 
    <label for="edu">Education: </label>
<select id="edu">
  <option value="-1">Select...</option>
  <option value="0">Primary school</option>
  <option value="1">Secondary school</option>
  <option value="2">Higer education</option>
</select>
  • [With multiple attribute]: 
    <label for="edu">Education: </label>
<select multiple>
  <option value="0">banana</option>
  <option value="1">apple</option>
  <option value="2">potato</option>
  <option value="3">avocado</option>
</select>
  • not very intu√Øtive for the user; use checkboxes instead

File upload

  • File upload field: 
    <label for="photo">Profile picture: </label>
<input type="file" id="photo">
  • The multiple attribute allows multiple file uploads: 
    <label for="photo">Profile picture:</label>
<input type="file" id="photo" multiple>
  • The accept attribute limits file types: 
    <input type="file" accept="image/jpeg,image/jpg,image/png">
    <input type="file" accept=".jpeg,.jpg,.png">

Simple buttons

  • A basic button: 
    <input type="button" value="Click me">
  • Alternative syntax (offers more possibilities): 
    <button>
  <img src="img/07_forms/ok.png" alt="save icon"/> Save
</button>
<button>
  <img src="img/07_forms/cancel.png" alt="cancel icon"/> Cancel
</button>
  • these buttons do nothing by default and are only useful with Javascript

Submit buttons

  • To make a submit button, use the type="submit" attribute: 
    <input type="submit" value="Send">

<button type="submit">Confirm</button>
  • Always provide a submit button, even if you don't see one on the design:
    • submit buttons help screen readers; we will hide them later with CSS
If your <button> only contains an image, e.g. 
<button type="submit">
  <img src="img/icon_save.png" alt="save">
</button>
make sure to provide an alt attribute with the same value as the text on the image. This greatly helps screen readers.

  • Feel free to use these new input types, but remember

    • alternative: use a Javascript library or a polyfill (e.g. for date)

Button or link?

  • Links and buttons are often confused, because links sometimes look like buttons and vice versa. The difference is very clear:
    • inside a form (submit): use a button
    • outside a form (action or navigation): use a link
  • → Where would you use a button, and where a link?

Form attributes

Placeholder

  • Shows a helper text in the field: 
    <p>
  <label for="name">Your name:</label>
  <input type="text" id="name" placeholder="e.g. John Smith">
</p>

Autofocus

  • Focuses on the field when the form loads: 
    <p>
  <label for="login">Your login:</label>
  <input type="text" id="login">
</p>
<p>
  <label for="pw">Your password:</label>
  <input type="password" id="pw" autofocus>
</p>

Required

  • Will perform basic validation based on the input type on submitting. Use the formnovalidate attribute on a button to disable validation. 
    <form action="#">
  <p>               
    <label for="email">Your email:</label>
    <input type="email" id="email" required>
  </p>               
  <p>               
    <input type="submit" value="Go!">
    <input type="submit" value="Cancel" formnovalidate>
  </p>               
</form>
